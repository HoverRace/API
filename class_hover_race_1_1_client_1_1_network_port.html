const NetMessageBuffer * HoverRace::Client::NetworkPort::Poll ( int  pClientId,
BOOL  pCheckClientId 
)

Check if any new messages have been recieved.

If not, NULL is returned. Otherwise the first message in the queue is returned. The UDP socket is checked before the TCP socket (TCP messages are not as time-critical).

The parameter pClientId is a dirty hack for the one-port UDP hack. Since this polls the global UDP receive port it is possible we could receive a message not addressed to this particular port; therefore, we peek at the message and make sure it has the same ID as pClientId, or, we ignore it.

Parameters
pClientIdId of the client a packet should be from
pCheckClientIdEnable client ID checking.
